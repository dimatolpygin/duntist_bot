# CLAUDE.md — заруб_бот (ЛСП Город STL файлы)

Telegram-бот приёма заказов на файлы для цеха. Клиент шлёт файлы боту → бот собирает
их в один заказ → карточка + файлы уходят в закрытую группу «Город». Заказчик — Zurab.

## Старт сессии (обязательно)

Перед любой работой:
1. Прочитать `docs/STATUS.md` — текущий этап, активная работа, блокеры.
2. `git tag -l "stage-*"` и сверить с таблицей в STATUS. Тег без записи в таблице = STATUS устарел.
3. `git log --oneline -10` — что менялось последним.
4. Открыть `docs/07_ROADMAP.md` на текущем этапе — там открытые чекбоксы.

Методология — okplan: три слоя правды (STATUS.md → чекбоксы ROADMAP → git-теги `stage-N-done`).
Приоритет при расхождении: **тег > чекбоксы > проза STATUS**.

## Правила ведения

- Выполняем **по 1 этапу за раз**. Этап закрывает **человек** после живой проверки — Claude не ставит тег наперёд.
- Не отмечать чекбоксы и не ставить теги наперёд.
- Коммиты статуса отдельно от кода: `chore(status): start/close stage N`.

## Стек

Python + aiogram 3, Postgres 16 (**отдельная схема**, локальную БД/чужие схемы не трогаем),
Redis (FSM), Docker, polling, автомиграции, полный слой логирования в терминал.
База — шаблон `../sub_bot` (~90% инфраструктуры готово).
